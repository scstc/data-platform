# 数据工程 & 数据集 —— 原始需求

> 来源：`数据工程&数据集.xlsx`（原始需求文件，2026-05-26），由 markitdown 转换后整理。

| 大类 | 需求项 | 需求内容 | 评分方式 |
| --- | --- | --- | --- |
| 数据工程 | 数据接入--接入方式 | 1、支持对接S3对象存储（OSS/OBS）和hdfs分布式文件系统拉取文件<br>2、支持手动上传本地文件<br>3、支持直连数据库拉取数据（达梦，GoldenDB，kingbase，gaussdb，hologres，SDB巨杉，hive，doris）<br>4、支持外部服务通过API接口推送数据（承诺支持即可）<br>5、支持配置数据采集任务，批量从数据源拉取数据 | 按支持项完整性评分 |
| 数据工程 | 数据接入--数据类型 | 1、多模态数据（语音，图片，视频，文本）以及跨模态数据集<br>2、结构化数据与非结构化数据<br>3、COT思维数据<br>4、问答对数据<br>5、偏好数据<br>6、时序数据<br>7、GIS位置<br>8、不同类型数据融合，对外提供统一语义 | 按支持项完整性评分 |
| 数据工程 | 数据接入--数据格式 | 1、支持的常见的文件格式：TXT，pdf，ppt,doc,docx, xlsx，xls,csv,tsv,html, jsonl | 按支持项完整性评分 |
| 数据工程 | 数据安全--数据内容安全 | 1、支持利用大模型等技术对数据内容进行审核（黄赌毒政恐），避免数据投毒，支持自定义规则和敏感数据，并可以对有毒内容进行打标或者直接删除（需要有记录） | 通过性指标 |
| 数据工程 | 数据安全-数据访问安全 | 1、具有严格数据权限管理，无越权访问风险(通过性指标) | 通过性标准 |
| 数据工程 | 数据质量评估 | 1、支持利用大模型（自定义提示词等）或者内置算子或者自定义算子，对数据（所有的数据类型）进行评估打分，可以查看每条数据的得分详情<br>2、可生成质量分析报告，数据质量的评估报告需要支持可视化展示（可承诺支持）<br>3、支持一键删除低质数据（根据得分，用户可配置阈值），（承诺支持即可），存储为新数据集或者新版本 | 按支持项完整性评分 |
| 数据工程 | 数据加工--数据清洗 | 1、支持通过大模型，内置算子，或者自定义算子，对数据进行过滤清洗<br>2、内置算子应包含：支持移除不可见字符、规范化空格、去除乱码、繁体转简体、去除网页标识符、去除表情开关，数据去重等<br>3、支持对清洗结果进行预览，根据预览确定是否采纳并持久化存储 | 按支持项完整性评分 |
| 数据工程 | 数据加工--数据合成与增强 | 1、支持按照要求利用大模型等技术，内置算子，或者自定义算子自动生成数据（承诺支持即可，已有可加分）<br>2、支持按照要求利用大模型等技术，内置算子，或者自定义算子自动对已有数据进行增强。<br>3、支持数据蒸馏<br>4、支持对合成或者增强，蒸馏的结果进行预览以及持久化存储，可以区分哪些是原始数据，哪些是合成数据 | 按支持项完整性评分 |
| 数据工程 | 数据加工--数据处理 | 1、支持利用大模型，内置算子，自定义算子对原始数据进行加工生成新的数据集，新数据集支持预览以及持久化存储<br>2、内置算子应该包含COT数据生成，QA对抽取等 | 按支持项完整性评分 |
| 数据工程 | 数据任务 | 1、有任务广场，可以查看数据处理，分析类的任务列表以及任务详情<br>2、任务支持重跑，停止，删除操作<br>3、支持通过ID，任务名称等信息进行检索<br>4、支持拖拉拽方式编排算子流程，可一键执行完整的数据清洗，加工等工作 | 按支持项完整性评分 |
| 数据工程 | 数据血缘 | 1、可以查看数据集的来源任务，以及任务所涉及到的所有资产信息，保证数据集可溯源，可复现 | 按支持项完整性评分 |
| 数据工程 | 数据标注 | 1、支持单人标注<br>2、支持智能标注与智能审核，人工审核<br>3、支持多人标注与多人审核，支持自动分配任务与人工分配任务，支持配置交叉比例<br>4、支持查看标注进度，标准结果，标注结果自动生成可视化报告 | 通过性指标 |
| 数据集仓库 | 元信息管理（承诺实现也可） | 1、固化信息：数据来源，类型，大小，创建人，创建时间，数据版本，更新时间，最后变更人等信息，<br>2、可变信息：名称，描述，有效时间等 | 是否满足，或是否承诺满足 |
| 数据集仓库 | 数据集打标 | 1、支持对数据集打标，随时增删改查数据集标签<br>2、支持配置数据标签库，根据要求增删改查库内标签内容<br>3、支持分析数据内容自动打标 | 按支持项完整性评分 |
| 数据集仓库 | 分级分类管理 | 1、支持按层级，类型对数据集进行分级分类管理 | 按支持项完整性评分 |
| 数据集仓库 | 检索与预览 | 1、检索：支持通过数据集名称，创建人，创建时间，数据类型，数据标签检索检索数据<br>2、支持根据数据内容对数据进行检索<br>3、列表展示（卡片展示）：批量展示数据集的名称，标签，描述，类型等信息<br>4、数据预览：支持预览具体数据 | 按支持项完整性评分 |
| 数据集仓库 | 版本 | 1、同一个数据集，可以有多个数据版本，可以查看不同数据版本的数据，每个版本均有各自的元数据 | 按支持项完整性评分 |
| 数据集仓库 | 数据托管 | 1、支持托管存在三方的S3存储上的数据，对上面的数据可以实现与被系统管理的数据相同的操作（删除等涉及到数据安全的操作除外） | 按支持项完整性评分 |
| 数据集仓库 | 支持的操作 | 1、支持数据集级别的删除与批量删除操作<br>2、支持数据级别的删除与批量删除操作<br>3、支持对于数据的增删改查操作，可基于操作结果生成新的版本或者新的数据集<br>4、数据集支持共享，共享的数据集权限可配置<br>5、数据集或者数据支持下载，导出至本地或者S3存储上<br>6、支持数据集生命周期管理，数据设置有效期，到期清理机制 | 按支持项完整性评分 |
| 数据集仓库 | 预置数据集 | 1、平台内置数据集： 适配于银行业场景，每个数据集的数量应不低于2000条 | 按支持项完整性评分 |
