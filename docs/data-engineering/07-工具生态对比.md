# 07 · 工具生态对比

> 选型结论先行：本项目已选 data-juicer 作为处理引擎（fork 在本仓库 `data-juicer/`），这个选择与调研结论一致——它和 NeMo Curator 是当前两大主力，且对中文/国内生态最友好。本篇帮你理解「为什么」以及「别的工具长什么样」。

## 主力对比

### Data-Juicer（阿里通义）——本项目选型（核验 3-0 ✓）

来源：[GitHub](https://github.com/datajuicer/data-juicer) · [2.0 论文 arXiv:2501.14755](https://arxiv.org/abs/2501.14755)（NeurIPS'25 Spotlight）

- **背景**：阿里通义实验室发起，联合阿里云 PAI、Anyscale（Ray 团队）、NVIDIA NeMo 团队、中山大学共同开发，Apache 2.0
- **算子体系**（写作时实测 212 个，README 称 200+，**活体增长值，以 [docs/Operators.md](https://github.com/modelscope/data-juicer/blob/main/docs/Operators.md) 为准**）：

| 算子类型 | 数量 | 职责 |
|---------|------|------|
| Formatter | 8 | 加载数据集、格式统一 |
| Mapper | 123 | 编辑/转换样本（清洗、合成都在这） |
| Filter | 57 | 计算统计量并按规则移除样本 |
| Deduplicator | 10 | 找冗余（MinHash/SimHash/精确） |
| Selector | 5 | 按偏好排序采样 |
| 其他 | 9 | Aggregator/Grouper/Pipeline |

- **多模态**：文本/图像/音频/视频全覆盖；**分布式**：Ray 集成处理 TB 级
- 本仓库实操：`/dj-demo`（CLI）、`/dj-web`（网页）、`/dj-api`（REST，985 个接口实测）

### NVIDIA NeMo Curator（核验 3-0 ✓）

来源：[官方页](https://developer.nvidia.com/nemo-curator) · [技术博客](https://developer.nvidia.com/blog/mastering-llm-techniques-data-preprocessing/)

- 开源、可扩展，覆盖文本/图像/视频/音频（25.09 起转 Ray 架构，旧资料讲 Dask 的已过时）
- 文本管线**六阶段**：①下载抽取为 JSONL → ②初步清洗（Unicode/语言分离）→ ③启发式过滤 → ④去重（精确/模糊/语义）→ ⑤模型质量过滤 + PII 脱敏 + 分布式分类 → ⑥混合与打散
- 定位差异：GPU 加速是卖点，与 NVIDIA 训练栈耦合更紧

## 其他值得知道的

| 工具 | 定位 | 一句话 |
|------|------|--------|
| [datatrove](https://github.com/huggingface/datatrove)（HuggingFace） | 预训练语料管线 | **FineWeb 就是用它做的**，大规模 web 数据处理参考实现 |
| [dolma](https://github.com/allenai/dolma)（AllenAI） | 语料工具包 | OLMo 开放模型的配套数据管线，透明可复现典范 |
| [Dataverse](https://github.com/UpstageAI/dataverse)（Upstage） | Spark 版 ETL | 基于 Apache Spark 多节点分布式（改配置即扩展），内置 PII 移除（核验 3-0 ✓） |
| [Label Studio](https://labelstud.io/) | 标注平台 | 开源标注事实标准，data-juicer 仓库有集成迹象（待补研） |
| [Presidio](https://github.com/microsoft/presidio/)（Microsoft） | PII 引擎 | NER+规则混合识别与脱敏（见 [05](05-数据安全与合规.md)） |

> ⚠️ **被驳回的说法**（1-2 票）：「Dataverse 支持清洗/去重/增强/去污染/去偏/去毒/PII 完整全套」——过度宣称，经核验确证的只有 ETL+Spark+PII。引用工具能力时小心营销话术，以仓库代码为准。

## 选型思考框架（给零基础读者）

1. **处理引擎**（清洗/去重/过滤/合成）：data-juicer ✅ 已选，算子最全、中文生态好
2. **GPU 重负载**（语义去重/大规模分类）：可借鉴 NeMo Curator 思路，data-juicer 的 Ray 模式也能跑
3. **标注**：Label Studio（先验证 data-juicer 现成集成）
4. **PII**：Presidio 思路 + 中文实体扩展，或 data-juicer 内置脱敏 Mapper
5. **数据集仓库/血缘/权限**：没有现成开源全家桶，自研增量在这（见 [06](06-数据标注与数据集管理.md)、[08](08-需求映射.md)）
