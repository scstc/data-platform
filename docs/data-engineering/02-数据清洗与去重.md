# 02 · 数据清洗与去重

## 第一部分：数据清洗

清洗解决的是「单条数据内部的脏」：乱码、不可见字符、HTML 残留、表情、繁简混排等。

### 典型清洗算子（对照 data-juicer 内置 Mapper）

- 移除不可见字符 / 规范化空格
- 去除乱码、去除网页标识符（HTML 标签）
- 繁体转简体、去除表情
- Unicode 规范化、语言识别分流（fastText `lid.176.bin`，本仓库 demo 实测过：6 条中英法混合样本过滤后剩 2 条中文）

清洗一般是**规则算子**（代码而非模型）——便宜、可解释、可复现。模型只在规则搞不定时上（如语言识别用 fastText）。

## 第二部分：数据去重

### 为什么去重是独立关键阶段

重复数据浪费算力、导致模型记忆（memorization）、降低多样性。业界把它从「清洗」中独立出来，因为方法体系完全不同。

### 三/四分法（多源相互印证，核验 3-0 ✓）

来源：[Data×LLM](https://arxiv.org/html/2505.18458v3) · [NVIDIA 技术博客](https://developer.nvidia.com/blog/mastering-llm-techniques-data-preprocessing/) · [Data Management Survey](https://arxiv.org/html/2312.01700v2)

| 类型 | 原理 | 代表方法 | 成本 |
|------|------|---------|------|
| **精确去重** | 内容完全相同 → hash 相同 | MD5 分桶、Suffix Array | 最低 |
| **模糊去重** | 近似重复（改几个词） | **MinHash + LSH**、SimHash | 中 |
| **频率分析** | 软去重，按频率降权 | SoftDeDup | 中 |
| **语义去重** | 意思相同但表述不同 | SemDeDup（embedding + k-means + 余弦阈值） | 最高（要算 embedding） |

### MinHash 是什么（最常用，必须懂）

把每篇文档拆成 N-gram 集合，用 Jaccard 相似度 `J(X,Y) = |X∩Y| / |X∪Y|` 衡量两篇文档的重合度。MinHash 用一组哈希函数把这个相似度估计变成可大规模并行的近似计算，再用 LSH（局部敏感哈希）分桶避免两两比较。

**业界实例——FineWeb 的参数**（[arXiv:2406.17557](https://arxiv.org/html/2406.17557v1)，核验 3-0 ✓）：

- 5-grams，112 个 hash 函数，分 14 桶 × 每桶 8 个 hash
- 目标：捕捉相似度 ≥75% 的文档对；任一桶内 8 个 MinHash 全相同即判重

### 实证教训：去重不是越狠越好

FineWeb 发现（核验 3-0 ✓）：**按快照逐个去重（per-snapshot）效果优于跨全库全局去重**。全局去重对最旧快照移除了约 90% 数据，但留下的 10% 质量反而比删掉的 90% 更差。

> ⚠️ 注意限定：「移除 90%」只针对最旧快照，不是所有数据。教训的本质是：全局激进去重会系统性偏向保留某类残次内容，去重粒度要做实验验证。

### 语义去重的价值

[Data Management Survey](https://arxiv.org/html/2312.01700v2)（核验 3-0 ✓）：N-gram+hashing 是最常用技术，但**神经/语义方法（如 SemDeDup）能移除嵌入空间中靠近的语义重复，显著超越传统方法**——代价是 embedding 计算开销。
