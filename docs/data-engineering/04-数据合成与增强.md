# 04 · 数据合成与增强

> 核心问题：真实数据不够、太贵、或太敏感（金融场景常见）怎么办？用 LLM 造。

## 三代指令合成方法（按演进顺序）

### Self-Instruct（2022，[arXiv:2212.10560](https://arxiv.org/abs/2212.10560)）

从一小撮人写的**种子任务**出发，让 LLM 仿写生成新指令 + 答案，再过滤去重回灌。开创了「用模型自举指令数据」的路线（Alpaca 即此法产物）。

### Evol-Instruct（2023，[arXiv:2304.12244](https://arxiv.org/abs/2304.12244)，WizardLM）

不只仿写，而是**进化**：把已有指令系统性地改难（加约束、深化、具体化）和改广（换领域），多轮迭代得到复杂度梯度更丰富的指令集。

### Magpie（2024，[arXiv:2406.08464](https://arxiv.org/pdf/2406.08464)，核验 3-0 ✓）

最省事的一代：**连种子 prompt 都不要**。原理——对齐过的 LLM（如 Llama-3-Instruct）有固定的 chat 模板，只把模板喂到「user 消息该出现的位置」就停，模型会自回归地「脑补」出一条用户查询；再正常生成回复，即得一条 (指令, 响应) 对。

实战规模（核验 3-0 ✓）：对 Llama-3-Instruct 家族生成 **400 万条**指令+回复，质量筛选后留 **30 万条**高质量实例。

> 💡 三代方法共同的工程要点：**合成 → 质量过滤 → 去重**，合成数据必须再过一遍第 02/03 篇的流水线，垃圾合成数据比没有更糟。

## 知识蒸馏（Distillation）

让小模型学大模型：用强模型（教师）生成响应/推理链（如 COT 数据），训练弱模型（学生）。FineWeb-Edu 的「LLM 打标 → 小分类器」也是蒸馏思想在数据质量上的应用（见 [03](03-数据质量评估.md)）。

## 合成数据在金融等敏感行业的定位（核验 3-0 ✓，置信度 medium）

来源：[arXiv:2503.14023](https://arxiv.org/pdf/2503.14023)

> LLM 生成的合成数据可在标注数据**稀缺、昂贵或敏感**的场景下增强甚至替代真实数据集。

金融/银行业佐证（来自更广文献）：英国 FCA 合成数据沙箱用于反洗钱场景；有研究报告 AML 测试中合成数据达到 96-99% 效用等价。

> ⚠️ 同时必须知道**模型崩溃（model collapse）**风险：多轮「模型生成→训练→再生成」会导致分布退化（[arXiv:2404.05090](https://arxiv.org/abs/2404.05090) 等）。业界共识是「**扩充而非完全替代**」真实数据，并严格区分原始/合成数据来源——这正对应原始需求第 8 项「可以区分哪些是原始数据，哪些是合成数据」。

## 用什么模型（速查）

| 任务 | 推荐 |
|------|------|
| 指令合成 | 对齐好的开源强模型（Magpie 法零种子启动）或商用 API |
| COT 数据生成 | 推理强的教师模型，生成后用验证器/规则校验答案正确性 |
| 数据增强（改写/扩写） | 中档 LLM 即可，配合 Evol-Instruct 式提示词 |
| 合成质量过滤 | 奖励模型或 LLM-as-a-Judge（小样本）+ fastText/BERT（大规模） |

## 对照 data-juicer

data-juicer 含合成/增强类 Mapper（如基于 LLM 的 QA 抽取、文本增强算子），原始需求第 9 项「内置算子应包含 COT 数据生成、QA 对抽取」可部分由其覆盖，缺口部分自定义算子补齐——它的算子体系就是为这种扩展设计的。
